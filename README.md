# Software Requirements

**Functional Requirements (FRs):**

User Authentication

> *The system should allow administrators and staffs to log in securely using their name, email address and password. Users with "Inactive" status must be denied access.*

**Staff Management**

> *Administrators must be able to register new staff, update their details, and deactivate their accounts when they resign.*

**Tank Management**

> *Users should be able to register new tanks, defining their name, capacity and operational status.*

**Fish Batch Registration**

> *Users should be able to register a new batch of fish for a specific tank with recording the species, source stocking date and initial quantity.*

**Water Quality Logging**

> *Users must be ableto record daily water quality parameters for specific tank.*
    
**Critical Alert**

> *The system must automatically analyse waterlogs and trigger an alert on the dashboard when the water quality parameters are unsafe (eg. pH < 6.0).*
    
**Inventory Item Management**

> *Users should be able to add new types of feed or medicine to the inventory list.*
    
**Ubvebtiry Restocking**

> *Users must be able to update stock when new supplies arrive.*
    
**Feeding Recording**

> *Users should be able to record feeding activities. The system must automatically deduct the used amount from the inventory stock to keep accurate records.*
    
**Harvest Recording**

> *User must be able to record harvest details when a batch is sold, including total weight and revenue generated.*
     
**Real-time Dashboard**

> *The system should display an overview of all tanks, showing their current species and latest water quality status.*
     
**Performance Reporting**

> *The system must generate monthly reports summarising total feed consumption vs. total harvest revenue to calculate profit estimates.*
